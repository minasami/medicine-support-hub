# Pharmacy Finance Control Principles

The pharmacy module should treat finance and stock records as operational evidence, not disposable UI data.

## Principles

1. Current-period work can be entered and reviewed.
2. Historical reports should remain stable.
3. Corrections should be visible and explainable.
4. Branch owners should have clear accountability.
5. Accountants should be able to work efficiently without having unsafe powers.
6. Platform admins should support the system without becoming routine data-entry users.

## Product direction

The next finance maturity step is a monthly review and close process. Once a reporting period is reviewed, later changes should be handled through visible corrections rather than silent edits.

## Why this matters

This improves trust in:

- monthly sales reports,
- expense reports,
- supplier balances,
- stock movement history,
- branch accountability,
- audit readiness.

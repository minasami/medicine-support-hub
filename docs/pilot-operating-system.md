# Pilot Operating System

The Medicine Support Hub pilot layer provides one connected workflow for defining, launching, governing, measuring, and reporting a medicine-access pilot.

## Production modules

- Command Center: `/workspace/pilot-command/:programId`
- Pilot Workspace: `/workspace/pilots/:programId`
- Readiness Assessment: `/workspace/pilot-readiness/:programId`
- Executive Summary: `/workspace/pilot-executive/:programId`
- Governance Workspace: `/workspace/pilot-governance/:programId`
- Printable Pilot Report: `/workspace/pilot-report/:programId`
- Program Record: `/workspace/programs/:programId`
- Impact Reporting: `/impact`

## Data foundation

- `programs`
- `pilot_milestones`
- `pilot_deliverables`
- `pilot_sites`
- `pilot_decisions`
- `pilot_meetings`
- `pilot_readiness_summary`
- `pilot_executive_summary`

## Operating cadence

1. Define scope, objectives, sites, dates, budget, risks, and success criteria.
2. Assign milestones and deliverables with accountable owners.
3. Attach evidence and approve deliverables.
4. Review readiness before launch.
5. Record governance decisions, meetings, and actions.
6. Monitor delivery, enrollment, budget, and support-request outcomes.
7. Produce executive and printable reports for partners and funders.
